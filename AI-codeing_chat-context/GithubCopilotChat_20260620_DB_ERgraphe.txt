## 現状の把握

- 今のアプリは sampleData.ts の `VTuberProfile` モックデータを使っている
- 実際の永続化はまだなく、`AppContext` で状態管理している
- `ProfileFormPage` にある入力項目がそのまま DB に必要な項目の候補になる

> .design フォルダはデザイン/プロトタイプ系のアセットなので、DB構築には含めなくて良いです。

## テーブル構成の提案

### 1. `vtuber_profiles`
プロフィールの基本情報を格納する主テーブル

- `id UUID PRIMARY KEY`
- `name VARCHAR NOT NULL`
- `nickname VARCHAR`
- `affiliation VARCHAR`
- `birthday VARCHAR` または `TEXT`
- `debut VARCHAR`
- `blood_type VARCHAR`
- `height VARCHAR`
- `favorite_things TEXT`
- `disliked_things TEXT`
- `hobby TEXT`
- `catchphrase TEXT`
- `dream TEXT`
- `message TEXT`
- `one_word TEXT`
- `activity_history VARCHAR`
- `activity_genre VARCHAR`
- `activity_status VARCHAR`
- `image_url TEXT`
- `free_description TEXT`
- `view_count INT DEFAULT 0`
- `like_count INT DEFAULT 0`
- `created_at TIMESTAMP WITH TIME ZONE DEFAULT now()`
- `updated_at TIMESTAMP WITH TIME ZONE DEFAULT now()`

---

### 2. `profile_tags`
タグを正規化して検索しやすくする

- `id SERIAL PRIMARY KEY`
- `profile_id UUID REFERENCES vtuber_profiles(id) ON DELETE CASCADE`
- `tag VARCHAR NOT NULL`
- `sort_order INT`（任意）
- `UNIQUE(profile_id, tag)`

> いきなり `tags` を JSON 配列にしても良いが、検索・絞り込みを考えるなら別テーブルが安全。

---

### 3. `profile_sns_links`
SNSリンクの個別管理

- `id SERIAL PRIMARY KEY`
- `profile_id UUID REFERENCES vtuber_profiles(id) ON DELETE CASCADE`
- `icon VARCHAR`（`youtube` / `x` / `tiktok` / `globe` など）
- `label VARCHAR`
- `url TEXT NOT NULL`
- `sort_order INT`

---

### 4. `profile_videos`
動画リンクを個別管理

- `id SERIAL PRIMARY KEY`
- `profile_id UUID REFERENCES vtuber_profiles(id) ON DELETE CASCADE`
- `url TEXT NOT NULL`
- `sort_order INT`

---

## 追加で考えたいテーブル

### 5. `users`
将来的にログイン/ユーザー機能を入れるなら

- `id UUID PRIMARY KEY`
- `name VARCHAR`
- `email VARCHAR UNIQUE`
- `provider VARCHAR`（Google / GitHub / guest など）
- `created_at TIMESTAMP`
- `updated_at TIMESTAMP`

### 6. `profile_likes`
ユーザーごとのいいね履歴を残すなら

- `id SERIAL PRIMARY KEY`
- `profile_id UUID REFERENCES vtuber_profiles(id) ON DELETE CASCADE`
- `user_id UUID REFERENCES users(id)` または `session_id`
- `created_at TIMESTAMP`

> 現状は `likeCount` と `likedProfileIds` だけなので、最初は `like_count` のみでも十分。将来の拡張で `profile_likes` を追加するのが良い。

  - VTuber の基本プロフィールを保持

  - タグを正規化して管理

- `profile_sns_links`
  - YouTube / X / TikTok / Web などの外部リンクを個別管理

- `profile_videos`
  - 動画リンクを個別管理
  - 最大12件の動画リンクにも対応しやすい

- `users`
  - 将来ログイン機能を入れる場合に利用
  - 現状は不要だが、拡張設計として有用

- `profile_likes`
  - ユーザー別の「いいね」履歴を残すなら
  - 単純な `like_count` だけでなく UX 拡張が可能
